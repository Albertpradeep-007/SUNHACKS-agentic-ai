Complete tech stack

Data ingestion layer
PyGithub / python-gitlab PyDriller gitpython
PyDriller is the star — it walks every commit, extracts diff stats, file churn, author info, added/removed lines. No need to clone manually. python-gitlab / PyGithub handle REST auth + webhook setup.
from pydriller import Repository for commit in Repository(".", since=..., to=...).traverse_commits(): commit.modified_files # list of changed files + metrics
Code quality analysis
Radon pylint / flake8 coverage.py
Radon computes cyclomatic complexity (CC), Halstead metrics, and maintainability index per file. Run it on every snapshot to build a time-series of quality.
radon cc src/ -a -s # complexity grades per function radon mi src/ -s # maintainability index 0–100
AI / orchestration layer

LangGraph gives you state-machine-style multi-agent flows. Each specialist agent (see Agents tab) is a LangGraph node. Use tool-calling to pass structured JSON between nodes.

FastAPI serves the dashboard + triggers analysis jobs. SQLite is fine for hackathon. Store metrics as time-series rows per (repo, file_path, commit_hash).

Multi-agent architecture (LangGraph nodes)

Agent 1
Repo Miner
Uses PyDriller to extract: commit history (last 90 days), files changed per commit, bug-fix commits (detect via commit message regex: fix/bug/patch/issue), test file ratio, deployment-tagged commits. Outputs structured JSON per module.
Agent 2
Health Scorer
Takes mined data, runs Radon complexity on current HEAD, computes a weighted health score per module: churn×0.3 + bug_density×0.3 + complexity×0.25 + coverage_drop×0.15. Outputs score 0–100 + risk tier (green/yellow/red).
Agent 3
Risk Predictor
LLM + trend analysis. Takes time-series of health scores, finds accelerating degradation patterns, predicts which modules will hit "critical" in next 90 days. Can use simple linear regression + LLM interpretation, or a pre-trained sklearn model (gradient boosting on commit features).
Agent 4
Report Writer
LLM agent (GPT-4o / Claude). System prompt: "You are a CTO writing a 1-page business report for a non-technical CEO." Input: risk tiers, module names, estimated eng-hours to fix, severity. Output: cost-of-inaction ($), recommended priority order, plain-English narrative. No jargon.
LangGraph flow
user_input (repo URL + token) → repo_miner_node → commit_metrics JSON → health_scorer_node → module_scores JSON → risk_predictor_node→ risk_predictions JSON → report_writer_node → CEO report (markdown → PDF)
